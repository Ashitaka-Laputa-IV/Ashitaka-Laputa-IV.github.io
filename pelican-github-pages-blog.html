Pelican + GitHub Pages = 个人博客

实现思路

以 Markdown（支持其他格式）作为博客编写的原始格式，通过 Pelican 静态网站生成器将其转换为静态网页，然后上传到 GitHub Pages 进行自动托管。

依赖环境

  • Python（Pelican 是 Python 的模块）

详细步骤

1. 安装 Pelican

python -m pip install "pelican[markdown]"

2. 初始化 Pelican 环境

在一个新文件夹（建议新建一个仓库）中运行：

pelican-quickstart

按照提示填写相关配置信息。

3. 编写博客文章

在 content 目录下创建 Markdown 文件，需要包含以下元数据：

Title: 文章标题
Date: 2026-02-27 14:11
Category: 分类
Tags: 标签1, 标签2
Slug: 文章URL路径
Author: 作者
Summary: 文章摘要

## 正文内容
这里是文章的具体内容...

4. 生成静态网页

运行以下命令将内容转换为静态网页，输出到 output 文件夹：

pelican content

5. 创建 GitHub 仓库

在 GitHub 上创建名为 yourname.github.io 的仓库（严格按照此命名格式），并且必须设置为公开（public）。

6. 配置 GitHub Pages

在仓库的 Settings -> Pages 选项中，将 Branch 设置为 master，Root 设置为 /（根目录）。

7. 部署到 GitHub Pages

在 output 文件夹中执行以下命令：

git init
git remote add origin https://github.com/yourname/yourname.github.io
git add .
git commit -m "Initial commit"
git push -u origin master

8. 访问博客

部署完成后，你可以通过 https://yourname.github.io 访问你的个人博客。

注意事项

  • 确保你的 Markdown 文件包含正确的元数据格式
  • 每次更新博客后，需要重新生成静态网页并推送到 GitHub
  • 可以通过修改 pelicanconf.py 文件来自定义博客的配置
